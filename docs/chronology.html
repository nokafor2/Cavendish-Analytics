Phase 0 — 3 Days Before Week 1

Reverse Proxy Foundations (k3s)

Build manually on a single EC2 instance. Every component has a direct EKS equivalent — see docs/phase0-to-eks-mapping.md.

Day 1 — k3s + Nginx Ingress + DNS P0-D1
  • Launch EC2 (t3.small), open ports 22, 80, 443, 6443
  • Install k3s with Traefik disabled
  • Helm install Nginx Ingress Controller → confirm IngressClass: nginx
  • Register FreeDNS subdomain: api.yourname.mooo.com → EC2 public IP
  • Deploy placeholder app + Ingress with Host rule

Execution — Day 1

Prerequisites: FreeDNS account (Afraid.org), kubectl and helm on your local machine. Replace <EC2_PUBLIC_IP>, your-key.pem, and api.yourname.mooo.com with your values.

Step 1 — Launch EC2

Create a t3.small (Ubuntu 22.04). Security group inbound: 22 (SSH), 80 (HTTP), 443 (HTTPS), 6443 (k3s API). Note the public IP.

Step 2 — SSH into EC2 and install CLI tools
ssh -i your-key.pem ubuntu@<EC2_PUBLIC_IP>

# kubectl
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
sudo install -o root -g root -m 755 kubectl /usr/local/bin/kubectl

# helm
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
Step 3 — Install k3s (Traefik disabled)
curl -sfL https://get.k3s.io | INSTALL_K3S_EXEC="--disable traefik" sh -

mkdir -p ~/.kube
sudo cp /etc/rancher/k3s/k3s.yaml ~/.kube/config
sudo chown $(id -u):$(id -g) ~/.kube/config

kubectl get nodes
# Expected: one node in Ready state
Step 4 — Install Nginx Ingress Controller
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm repo update

helm install ingress-nginx ingress-nginx/ingress-nginx \
  --namespace ingress-nginx --create-namespace

kubectl wait --namespace ingress-nginx \
  --for=condition=ready pod \
  --selector=app.kubernetes.io/component=controller \
  --timeout=120s

kubectl get ingressclass
# Expected: nginx with controller ingress-nginx/ingress-nginx
Step 5 — Register FreeDNS subdomain

At freedns.afraid.org, create an A record: api.yourname.mooo.com → EC2 public IP. Wait 5–15 minutes for propagation.

nslookup api.yourname.mooo.com
# Expected: resolves to your EC2 public IP
Step 6 — Clone repo and deploy placeholder app
git clone <your-repo-url> Cavendish-Project
cd Cavendish-Project/phase0

kubectl apply -f manifests/http-echo-deployment.yaml
kubectl get pods
# Expected: http-echo pod Running

Manifest: phase0/manifests/http-echo-deployment.yaml

apiVersion: apps/v1
kind: Deployment
metadata:
  name: http-echo
  namespace: default
spec:
  replicas: 1
  selector:
    matchLabels:
      app: http-echo
  template:
    metadata:
      labels:
        app: http-echo
    spec:
      containers:
        - name: http-echo
          image: hashicorp/http-echo:1.0.0
          args: ["-text=Cavendish Phase 0 — routed from inside the cluster"]
          ports:
            - containerPort: 5678
---
apiVersion: v1
kind: Service
metadata:
  name: http-echo
  namespace: default
spec:
  selector:
    app: http-echo
  ports:
    - port: 80
      targetPort: 5678
Step 7 — Create Ingress with Host rule

Replace api.yourname.mooo.com with your FreeDNS subdomain before applying.

# Edit manifests/http-echo-ingress.yaml — set your host, then:
kubectl apply -f manifests/http-echo-ingress.yaml
kubectl get ingress
# Expected: ADDRESS column shows EC2 IP (may take ~30s)

Manifest: phase0/manifests/http-echo-ingress.yaml

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: http-echo
  namespace: default
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  ingressClassName: nginx
  rules:
    - host: api.yourname.mooo.com  # Replace with your FreeDNS subdomain
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: http-echo
                port:
                  number: 80
Step 8 — Verify deliverable P0-D1
curl http://api.yourname.mooo.com
# Expected: HTTP 200 — body contains "Cavendish Phase 0 — routed from inside the cluster"

kubectl get ingress
# Expected: ADDRESS populated with EC2 IP

curl -I http://api.yourname.mooo.com
# Expected: HTTP/1.1 200 OK
Verify: curl http://api.yourname.mooo.com returns HTTP 200. kubectl get ingress shows Address populated with EC2 IP.
Day 2 — cert-manager + Let's Encrypt HTTPS P0-D2
  • Helm install cert-manager with CRDs
  • Apply ClusterIssuer: phase0/manifests/cluster-issuer.yaml
  • Update Ingress with TLS block: phase0/manifests/http-echo-ingress-tls.yaml
  • Open port 80 on security group (HTTP-01 challenge requirement)
  • Watch challenge: kubectl get challenges -n default -w

Verify: kubectl get certificate shows Ready: True. curl -I https://api.yourname.mooo.com returns 200 with no cert warning.
Day 3 — IP Restriction + Decommission Pattern P0-D3
  • Apply whitelist annotation: phase0/manifests/http-echo-ingress-whitelist.yaml
  • Test from VPN/mobile data → expect HTTP 403
  • Test from allowed IP → expect HTTP 200
  • Delete Ingress manifest → subdomain returns 404
  • Restore manifest → returns 200 again
  • Commit timestamped curl output to docs/phase0-demo.txt
  • Write Phase 0 → EKS mapping paragraph in docs
Verify: Non-whitelisted IP → 403. Delete Ingress → 404. Restore → 200.

How Each Resource Is Developed

Detailed development approach per resource layer. Full guide: docs/architecture.md

Terraform (terraform/)
  1. Bootstrap S3 + DynamoDB for remote state
  2. VPC module — 3 AZs, public/private subnets, NAT
  3. EKS cluster v1.29 with managed node group
  4. ECR repos, S3 buckets, Route 53, ACM, Secrets Manager
  5. IRSA roles in irsa.tf — OIDC trust per ServiceAccount
  6. GitHub Actions OIDC provider for CI/CD
Helm Chart (chart/)
  1. values.yaml — base service definitions (range loop source)
  2. values.staging.yaml / values.production.yaml — env overrides
  3. deployment.yaml — range loop over .Values.services
  4. ingress.yaml — ALB annotations, ACM cert ARN, ExternalDNS hostname
  5. statefulset.yaml — PostgreSQL + postgres-exporter sidecar
  6. networkpolicy.yaml — default-deny + 4 allow rules
  7. hpa.yaml — conditional on values.hpa.enabled
ArgoCD (argocd/)
  1. Install ArgoCD via Helm in argocd namespace
  2. ApplicationSet with list generator — staging + production
  3. Each Application points to chart/ with env-specific values file
  4. Automated sync with prune + selfHeal enabled
  5. Add new environment = one line in generators list
Observability (monitoring/)
  1. kube-prometheus-stack Helm install
  2. ServiceMonitors label-matched to release
  3. postgres-exporter sidecar exposes pg_stat_activity
  4. Grafana dashboard JSON — 4 required panels
  5. PrometheusRules — API error rate + DB connections
Security Layer
  1. IRSA — no IAM users, no access keys in Git
  2. Secrets Manager + CSI Driver — credentials never in cluster YAML
  3. NetworkPolicy — flat networking replaced with explicit paths
  4. RBAC — named SA + Role + RoleBinding per Pod
  5. Pod Security Standards — restricted profile on namespaces
Disaster Recovery (velero/)
  1. Velero Helm install with S3 backend via IRSA
  2. BackupSchedule — daily 02:00 UTC, 30-day retention
  3. pg_dump CronJob — every 6 hours to S3 via IRSA
  4. Test: delete namespace → Velero restore → pg_dump restore
  5. Document RTO in docs/dr-runbook.md
CI/CD (.github/workflows/)
  1. ci.yml — helm lint + template validation on PR
  2. ci.yml — docker build + ECR push on merge to main
  3. Both use OIDC — role-to-assume, no static credentials
  4. deploy.yml — helm upgrade --atomic --wait on CI success
  5. Environment protection for production deploys
Application (app/)
  1. FastAPI with /health, /metrics (Prometheus), portfolio endpoints
  2. CPU-intensive /api/v1/analytics/compute for HPA load testing
  3. DB credentials from env vars (Secrets Manager via CSI)
  4. Non-root user, read-only root FS in Dockerfile
  5. ECR image scanned on every push
